## 1. 비주얼 테마

현대벤디스는 기업 식대·복지 결제를 다루는 B2B 서비스다. 화면은 신뢰와 명료함이 우선이다: 흰 배경 위에 차분한 벤디스 그린(#43A047)이 절제된 포인트로 쓰이고, 본문은 진회색(#212121), 구조는 보더와 옅은 배경으로 구분한다. 화려한 그라디언트, 장식용 그림자, 채도 높은 색 조합을 쓰지 않는다. 모서리 기본 8px의 부드러운 라운드가 전체 인상을 만든다.

## 2. 색상 팔레트와 역할

- 벤디스 그린(#43A047)은 세 가지 역할에만 쓴다: (1) 주요 동작 버튼 (2) 활성·선택 상태 (3) 포커스 표시.
- 초록을 넓은 배경(워시)이나 장식으로 쓰지 않는다. 연한 강조가 필요하면 primary-tint(#E8F5E9)만 사용한다.
- 성공 상태는 primary를 재사용한다. 새로운 초록 값을 만들지 않는다 - 과거 서비스마다 다른 초록이 5종 생겼던 문제의 재발 방지 규칙이다.
- 본문에 순수 검정(#000)을 쓰지 않는다. 본문 #212121, 보조 #616161, 비활성 #9E9E9E.
- 그레이는 Material Gray 스케일만 사용한다. #dcdcdc, #ededf2 같은 임의 회색 금지.

## 3. 타이포그래피

- 서체는 Pretendard 하나만 쓴다. 다른 서체를 섞지 않는다.
- 굵기는 400(본문) / 500(라벨) / 600(카드 제목, 버튼) / 700(제목) 네 단계만.
- 제목은 굵기로 위계를 만들고, 색으로 위계를 만들지 않는다.
- 숫자가 세로로 정렬되는 곳(표, 금액)은 font-variant-numeric: tabular-nums.
- 모바일 제목은 스케일 표의 Mobile 값을 쓴다. 본문(14/16)은 두 환경 동일.

## 4. 컴포넌트 스타일

### 버튼
- 변형: fill(채움, 기본) / weak(연한 배경 + 진한 글자) / ghost(텍스트) / danger
- fill Primary: 배경 #43A047, 글자 #fff / hover #388E3C / pressed #2E7D32
- weak: 배경 #E8F5E9, 글자 #2E7D32 / hover 배경 #C8E6C9 - 보조 강조, 필터 선택 등
- Secondary: 배경 #fff, 보더 1px #E0E0E0, 글자 #424242 / hover 배경 #F5F5F5
- Ghost(텍스트): 배경 투명, 글자 #43A047 / hover 배경 #E8F5E9
- Danger: 배경 #D32F2F, 글자 #fff - 삭제·되돌릴 수 없는 동작에만
- 크기와 radius(크기 비례): sm 32/r8 · md 40/r8 · lg 48/r12 · 모바일 기본 48
- 상태 계약(모든 버튼 필수): default / hover / pressed / focus / loading / disabled
  - focus: 키보드 포커스 시 2px #43A047 링 (outline-offset 2px)
  - loading: 스피너 표시, 버튼 폭 유지, 중복 제출 방지. 비동기 동작(저장·확정·결제)에는 반드시 구현
  - disabled: 배경 #E0E0E0, 글자 #9E9E9E
- 한 화면에 fill Primary 버튼은 1개가 원칙

### 입력창
- 높이 D 40 / M 48 · radius 8 · 보더 1px #E0E0E0 · 패딩 좌우 12
- 라벨: 13px · 500 · 입력창 위 6px
- 포커스: 보더 #43A047 + 링 0 0 0 2px rgba(67,160,71,0.18)
- 오류: 보더 #D32F2F + 아래 12px 헬퍼 텍스트(#D32F2F). 오류 문구는 원인과 해결을 구체적으로
- 비활성: 배경 #F5F5F5, 글자 #9E9E9E
- 읽기 전용: 배경 #FAFAFA, 보더 유지, 글자 본문색 - 수정 불가지만 읽어야 하는 값
- 플레이스홀더: #9E9E9E. 라벨 대체로 쓰지 않는다

### 셀렉트
- 입력창과 동일 프레임 + 우측 화살표 아이콘. 모바일에서는 네이티브 셀렉트 허용

### 체크박스 · 라디오
- 박스 20px · 체크박스 radius 4, 라디오 원형 · 선택 시 primary 채움 + 흰 표시
- 터치 영역은 라벨 포함 최소 44px

### 카드
- 배경 #fff · 보더 1px #EEEEEE · radius 8 · 패딩 D 20 / M 16
- 그림자 없음이 기본. 클릭해서 뜨는 카드만 subtle

### 배지 · 칩
- 높이 24 · pill(full) · 12px 600 · weak 변형(연한 배경 + 진한 글자)이 기본형
- fill(진한 채움 + 흰 글자)은 강한 강조가 꼭 필요할 때만
- 성공 #E8F5E9/#2E7D32 · 중립 #F5F5F5/#616161 · 주의 #FFF3E0/#B45309 · 실패 #FDECEA/#D32F2F

### 탭
- 높이 44 · 비활성 글자 #616161 400 · 활성 #43A047 700 + 하단 2px primary 언더라인
- 색만으로 활성을 표시하지 않는다(굵기+언더라인 병행) · 모바일은 가로 스크롤

### 테이블
- 헤더: 배경 #F5F5F5 · 13px 600 #616161 · 높이 40
- 행: 높이 48 · 하단 보더 #EEEEEE · hover 배경 #FAFAFA
- 금액·숫자는 우측 정렬 + tabular-nums
- 모바일(767px 이하)에서는 카드 리스트로 전환한다: 각 행을 카드 하나로, 첫 열을 제목으로

### 모달
- 데스크톱: 중앙 정렬 · max-width 480 · radius 12 · prominent 그림자 · 딤 rgba(0,0,0,0.5)
- 버튼은 우측 정렬(취소가 왼쪽, 주요 동작이 오른쪽)
- 모바일: 바텀시트로 전환 - 하단 고정 · 풀폭 · 상단 radius 16 · 상단 그립 바 · 버튼 풀폭 세로

### 토스트
- 배경 #212121 · 글자 #fff 14px · radius 8 · 하단 중앙 · 3초 자동 소멸
- 문구는 과거형 한 문장("저장되었습니다"). 성공은 초록 점, 실패는 빨간 점 표시

### 아이콘
- 24x24 뷰박스 · stroke 2 · currentColor. 임의 SVG 금지, 세트에 같은 규격으로 추가
- 크기 16/20/24만 사용

### 스피너
- 원형 보더(연한 초록 트랙 + primary 헤드) · 크기 sm 16 / md 24 / lg 32
- 버튼 내부 로딩은 스피너 단독이 아니라 버튼의 loading 상태를 쓴다

### 스위치
- 트랙 40x24 pill · off #E0E0E0 / on primary · 썸 20 흰 원 + subtle 그림자
- 즉시 적용되는 설정에만. 제출이 필요한 선택은 체크박스

### 툴팁
- 배경 #212121 · 12.5px 흰 글자 · radius 4 · hover/focus 시 표시(fast 페이드)
- 한 줄 보조 설명만. 필수 정보·행동은 툴팁에 넣지 않는다

### 드롭다운 메뉴
- 흰 배경 · 보더 #EEEEEE · radius 8 · standard 그림자 · 항목 14px, hover #FAFAFA
- 파괴적 동작은 빨간 글자(danger)로 맨 아래 · 바깥 클릭·ESC로 닫힘

### 페이지네이션
- 버튼 32x32 · radius 8 · 현재 페이지는 tint 배경 + primary-active 600 + aria-current
- 7페이지 초과 시 현재 주변만 표시하고 줄임표 처리

### 아코디언
- 항목 구분 1px #EEEEEE · 헤더 높이 48 · 14.5px 600 · 셰브론 회전(fast)
- 여러 항목 동시 열림 허용

### 데이트피커
- 네이티브 input[type=date]를 입력창과 같은 프레임으로. 라벨·도움말·오류는 필드 문법 공통
- 커스텀 달력이 필요해지면 API 유지한 채 내부만 교체

### 파일 업로드
- secondary 소형 버튼 + 선택된 파일명 텍스트(13px 보조색) · 미선택 시 "선택된 파일 없음"
- 허용 형식·용량은 도움말로 미리 안내

### 폼 레이아웃
- 필드 세로 스택 16px 간격 · 하단 버튼 우측 정렬(취소 왼쪽) · 모바일 풀폭 세로 전환
- 검증 결과는 각 필드의 오류 문구로 표시. 폼 상단 요약만으로 끝내지 않는다

## 5. 레이아웃 원칙

- 모든 간격은 4px 그리드의 spacing 값만 사용한다.
- 데스크톱: 콘텐츠 최대 1200px 중앙 정렬 · 거터 24px. 모바일: 풀폭 · 거터 16px.
- 형제 요소 간격은 gap으로 만들고, 개별 margin 누적으로 만들지 않는다.
- 섹션 사이 32~48px, 카드 내부 요소 사이 12~16px.

## 6. 깊이와 그림자

- 화면은 기본적으로 평평하다. 구분은 보더(#EEEEEE)와 surface(#F5F5F5)로 한다.
- 그림자는 '떠 있는 것'에만: 드롭다운·토스트 standard, 모달 prominent.
- 장식 목적의 그림자, 색이 들어간 그림자 금지.

## 7. Do / Don't

Do:
- 초록은 주요 버튼·활성 상태·포커스에만
- 비동기 동작 버튼에는 loading 상태를 넣고 버튼 폭을 유지
- 연한 강조는 primary-tint(#E8F5E9)로
- 본문 #212121, 그레이는 Material 스케일만
- 모서리 기본 8px, 간격은 4px 그리드
- 오류 문구는 원인과 해결책을 구체적으로

Don't:
- 새로운 초록 값 생성 금지 (#46B476, #0BB656 등 과거 값 사용 금지)
- 순수 검정(#000) 본문 금지
- 임의 회색(#dcdcdc, #ededf2 등) 금지
- 한 화면에 Primary 버튼 2개 이상 금지
- UI에 이모지 사용 금지 (상태 표시는 배지·색 점으로)
- 장식용 그림자·그라디언트 금지

## 8. 반응형

- 브레이크포인트는 하나: 767px 이하 = 모바일, 768px 이상 = 데스크톱. 태블릿 전용 분기를 만들지 않는다.
- 터치 대상 최소 44px. 모바일 버튼·입력창 높이 48px.
- 전환 규칙: 테이블 → 카드 리스트 · 모달 → 바텀시트 · 주요 버튼 → 풀폭 세로 배치.
- 데스크톱에서 다열 그리드는 모바일에서 1열로 쌓는다.

## 9. 상태

- 로딩: 스켈레톤(#F5F5F5 블록, 최종 레이아웃과 같은 크기). 스피너는 버튼 내부 로딩에만.
- 빈 화면: 한 문장 설명(#616161) + 필요 시 다음 행동 버튼 1개. 일러스트 없이 담백하게.
- 오류: 원인 + 해결책을 한 문장으로. "오류가 발생했습니다" 단독 사용 금지.
- 성공: 토스트 과거형 한 문장.

## 10. 모션

- duration은 두 단계뿐: `--vd-motion-fast` 150ms(토스트, 페이드류) · `--vd-motion-base` 250ms(모달, 바텀시트). 임의의 duration 금지.
- easing은 하나: `--vd-ease-out` cubic-bezier(0.2, 0, 0, 1) — 빠르게 시작해 부드럽게 멈춤. 스프링·바운스·오버슈트 금지.
- 모션은 상태 변화를 설명할 때만 쓴다(등장·펼침·포커스). 장식용 금지.
- 적용된 등장 애니메이션: 토스트(8px 아래에서 페이드인), 모달(0.97→1 스케일+페이드), 바텀시트(하단 슬라이드업), 배경 딤(페이드).
- prefers-reduced-motion: reduce 시 모두 제거.

## 11. 보이스 & 톤

원칙 두 가지:
- 바로 답할 수 있게: 열린 질문 대신 비교 가능한 구체적 선택지를 준다. 사용자가 고민 없이 답할 수 있어야 한다.
- 효익 먼저, 요구는 나중: 정보 입력·동의·결제를 요구하기 전에, 그걸 하면 무엇이 좋아지는지 먼저 보여준다.

- 존댓말("-해 주세요", "-되었습니다"). 계정·결제 화면은 격식체 유지.
- CTA는 동사형("조회", "정산 확정", "저장"). 애매한 "확인" 남용 금지.
- 오류는 사과 대신 해결책: "사업자등록번호 10자리를 확인해 주세요" (O) / "죄송합니다. 오류가 발생했습니다" (X)
- 마케팅 수사("혁신적인", "최고의") 금지. UI 문구에 이모지 금지.

- "벤디스 디자인 시스템 기준으로 정산 내역 테이블 화면을 만들어줘. 모바일에서는 카드 리스트로 전환되게."
- "primary 버튼과 입력창으로 로그인 폼을 만들어줘. 오류 상태 포함."
- "정산 확정 버튼에 loading 상태를 넣어줘. 폭 유지, 중복 제출 방지."
- "정산 확정 모달을 만들어줘. 데스크톱은 중앙 모달, 모바일은 바텀시트로."

검증 체크리스트 (만든 화면이 시스템을 따르는지):
1. 초록이 버튼·활성·포커스 외에 쓰였는가? → 제거
2. 토큰에 없는 색·크기·그림자 값이 있는가? → 토큰으로 교체
3. 모바일에서 터치 대상이 44px 미만인가? → 확대
4. 본문에 #000이 있는가? → #212121로 교체
