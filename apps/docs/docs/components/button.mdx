# Button

한 화면에 fill(primary) 버튼은 1개가 원칙입니다.

## Variant

<div className="demo">
  <Button>저장</Button>
  <Button variant="weak">더 보기</Button>
  <Button variant="secondary">취소</Button>
  <Button variant="ghost">건너뛰기</Button>
  <Button variant="danger">삭제</Button>
  <Button disabled>비활성</Button>
</div>

```tsx
<Button>저장</Button>
<Button variant="weak">더 보기</Button>
<Button variant="secondary">취소</Button>
<Button variant="ghost">건너뛰기</Button>
<Button variant="danger">삭제</Button>
```

## Size

sm 32 / md 40(기본) / lg 48. 모바일 주요 동작은 lg를 씁니다.
lg는 radius도 12px로 커집니다.

<div className="demo">
  <Button size="sm">작게</Button>
  <Button>기본</Button>
  <Button size="lg">크게</Button>
</div>

## Loading

비동기 동작 중에는 스피너를 표시하고 클릭을 차단합니다. 버튼 폭은 유지됩니다.

<div className="demo">
  <LoadingDemo />
</div>

```tsx
<Button loading={saving} onClick={save}>
  {saving ? '저장 중' : '저장'}
</Button>
```

## Props

| prop | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `variant` | `'fill' \| 'weak' \| 'secondary' \| 'ghost' \| 'danger'` | `'fill'` | 시각적 위계 |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | 높이 32 / 40 / 48 |
| `loading` | `boolean` | `false` | 스피너 + 클릭 차단, `aria-busy` 적용 |

그 외 `button` 기본 속성을 모두 받습니다. `ref` 전달을 지원합니다.

## 문구 규칙

- 버튼 문구는 동사로 끝나는 행동입니다: "저장", "정산 확정". "확인", "예"처럼 맥락 없는 단어는 피합니다.
